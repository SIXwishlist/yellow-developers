Comment configurer la sécurité de votre site web.

## Restriction utilisateur

Si plusieurs personnes peuvent éditer votre site web, vous pouvez restreindre chaque [compte d'utilisateur](adjusting-system-fr#comptes-d-utilisateurs) de sorte que certaines parties du site seulement puissent être éditable. Ainsi, les utilisateurs ne peuvent éditer uniquement que leur page d'accueil. Ceci protège votre site web de toute modification à travers le navigateur web.

Vous pouvez [tester ce mode de restriction](/tests/user-restrictions).

## Mode Safe

Si plusieurs personnes peuvent éditer votre site web, activez le mode "safe" afin que seule la syntaxe [Markdown](markdown-cheat-sheet-fr) puissent être utilisée. Ouvrez le fichier `system/config/config.ini` et changez `ParserSafeMode: 1`. Ceci protège votre site de tout dommage.

Vous pouvez [tester le mode safe](/tests/safe-mode).

## Sécurité du serveur

Pensez à vérifier que votre serveur web supporte HTTPS et que la connexion Internet soit cryptée. Il est préférable que votre site soit accessible uniquement via HTTPS. Ceci protège la confidentialité de vos utilisateurs, mais n'est pas supporté par tout les hébergeurs web.

Vous aurez besoin d'un serveur avec TLS et de certificats valides.
